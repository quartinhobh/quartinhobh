/btw como automatizar print das páginas ?                                                                                                                                                                           
                                            
    Pra automatizar prints (screenshots) das páginas do site, três caminhos práticos:                                                                                                                                 
                                               
    1. Playwright (mais robusto, recomendado)                                                                                                                                                                         
    bun add -D @playwright/test
    bunx playwright install                                                                                                                                                                                           
    import { test } from '@playwright/test';                      
    const pages = ['/', '/listen', '/u/arthur', '/lojinha'];                                                                                                                                                          
    for (const path of pages) {                
      test(`screenshot ${path}`, async ({ page }) => {
        await page.goto(`http://localhost:5173${path}`);
        await page.waitForLoadState('networkidle');
        await page.screenshot({ path: `screenshots${path.replace(/\//g, '_')}.png`, fullPage: true });
      });
    }
    Roda em CI, vários viewports (devices['iPhone 13']), modo dark, etc.

    2. Puppeteer — mesma ideia, API parecida, só Chromium.

    3. browser-use (já mencionado no seu CLAUDE.md) — Python, bom se quiser que um agente navegue + tire print de fluxos complexos com decisões.

    Pra esse projeto especificamente, sugiro Playwright porque:
    - Já roda headless no GitHub Actions (cabe no preflight existente)
    - Suporta prefers-reduced-motion (importante por causa dos stickers e animações zine)
    - Captura múltiplos viewports numa rodada só (mobile/desktop)
    - Gera diff visual se você usar toHaveScreenshot() — vira regression test grátis

    Padrão típico de organização:
    - web/visual-tests/pages.spec.ts — lista de rotas + viewports
    - Output em web/visual-tests/__screenshots__/
    - Step novo no .github/workflows/deploy.yml rodando antes do deploy: se algum diff > threshold, falha o build

    Quer que eu monte isso? Posso fazer no estilo do preflight que você já tem (lista de rotas públicas, sem auth) e plugar como step opcional no workflow.
